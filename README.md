# sc171-ward-terminal-monitor
A lightweight SC171-based ward terminal network and system status monitoring system.
# 基于 SC171 的智慧病房终端监测系统

## 项目简介

本项目基于广和通 SC171 开发板实现一套轻量级智慧病房终端状态监测系统。系统运行在 SC171 的 Linux 环境中，通过 Python 自带 HTTP 服务在板端开启 Web 页面，电脑或手机在同一局域网下可通过“板子 IP:5000”访问监测界面。

系统可实时采集本机 IP、网络延迟、下载速率、上传速率、CPU 占用率、内存占用率和 CPU 温度等数据，并根据网络延迟、系统负载和温度阈值判断终端状态为正常、一般或异常。同时，系统会将监测数据自动记录到 `network_log.csv`，便于后续查看和分析。

## 主要功能

1. 实时显示 SC171 终端 IP 地址；
2. 实时检测网络延迟；
3. 实时计算网络下载速率和上传速率；
4. 实时显示 CPU 占用率；
5. 实时显示内存占用率；
6. 实时读取 CPU 温度；
7. 根据阈值判断终端状态；
8. 自动生成 CSV 日志文件。

## 系统运行环境

* 硬件平台：广和通 SC171 开发板
* 操作系统：Linux / AidLux 环境
* 编程语言：Python 3
* 依赖环境：无需第三方库，仅使用 Python 标准库
* 访问方式：浏览器访问 `http://板子IP:5000`

## 使用方法

进入项目目录：

```bash
cd /home/sc171_project/network_monitor
```

运行程序：

```bash
python3 network_monitor.py
```

程序启动后，终端会显示本机访问地址和局域网访问地址，例如：

```text
板子本机访问: http://127.0.0.1:5000
局域网访问:   http://192.168.x.x:5000
```

在电脑或手机浏览器中输入局域网访问地址即可查看监测页面。

## 应用场景

本项目可用于智慧病房、物联网网关、床旁监测终端、移动护理终端等边缘设备的基础运行状态监控。通过实时监测网络状态和系统资源占用情况，可辅助判断终端是否存在网络异常、负载过高或温度过高等问题。
